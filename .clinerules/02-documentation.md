# Documentation Rules

## Mandatory Documentation Updates
- After creating or modifying any source file,
  corresponding documentation MUST be created or updated.

## Documentation Location
- Documentation must be written in Markdown.
- Place documentation in:
  - /docs
  - or a module-specific docs/ directory.

## Documentation Content
Each documented module must describe:
- Purpose of the module
- Responsibilities
- Inputs and outputs
- Dependencies
- Limitations

## Documentation Reading Requirement
- Before modifying any file, read existing documentation
  related to that file or module.
- Changes must remain consistent with documented behavior.

## Changelog
- Any user-visible or architectural change
  MUST be reflected in CHANGELOG.md.
